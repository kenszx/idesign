# Design Critique — 5-Dimension Expert Review

## When to Use

- "Review this design"
- "Is this design good?"
- "Rate my design"
- "What can be improved?"

## 5-Phase Workflow

### Phase 0: Understand
Ask ONE question:
- "What's the design goal / context? (target users, brand constraints, timeline)"

### Phase 1: Analyze
Review across 5 dimensions:

| Dimension | Score 0-10 | Criteria |
|---|---|---|
| **Philosophical consistency** | | Does the design have a clear concept? Are choices intentional? |
| **Visual hierarchy** | | Is information properly prioritized? Is the reading flow clear? |
| **Execution detail** | | Are spacing, typography, colors precise? Any technical debt? |
| **Functionality** | | Does it work? Is the UX intuitive? Edge cases handled? |
| **Innovation** | | Does it bring something new? Or is it generic? |

### Phase 2: Report
For each dimension:
- Score (0-10) with brief justification
- Specific evidence from the design
- One concrete example of what works and what doesn't

### Phase 3: Action Items
Categorize into:
- **Keep** (3 things working well)
- **Fix** (3 things to improve)
- **Quick Wins** (low-effort, high-impact changes)

### Phase 4: Visual Summary
Generate an HTML radar chart showing scores across 5 dimensions.

### Phase 5: Iteration
Offer to implement the top 3 fixes.
Ask: "Shall I apply these changes?"

## Rules

- **Specific > Vague**: "The CTA button has 4.5:1 contrast, needs 4.5:1 minimum" not "make it more accessible"
- **Balanced**: Always include what's working, not just what's wrong
- **Actionable**: Every issue must have a concrete fix suggestion
