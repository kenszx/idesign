# Anti-AI-Slop Rules

These rules apply to ALL visual output across all Idesign sub-skills.
Read this file before generating any HTML/CSS/visual design.

## Visual Clichés to Avoid

- ❌ **Purple/blue gradient backgrounds** — Overused AI default
- ❌ **Emoji icons in UI** — Use SVG or simple geometric shapes
- ❌ **Rounded corners with left accent border** — Generic dashboard trope
- ❌ **Inter/Roboto as display type** — Reserved for body text only
- ❌ **SVG-drawn human faces** — Use photos or abstract placeholders
- ❌ **Card shadows everywhere** — Creates visual noise
- ❌ **"Hero section with floating elements"** — Lazy default pattern
- ❌ **Animated gradient text** — Distracting and amateurish

## Design Quality Standards

1. **Typography hierarchy** — Use at most 2 typefaces: one display + one body
2. **Color discipline** — Primary palette ≤ 3 colors. Neutrals don't count.
3. **Spacing** — Consistent rhythm. Use 4px/8px grid.
4. **Content > Decoration** — Every decorative element should serve a purpose
5. **Accessibility** — Contrast ratio ≥ 4.5:1 for body text
6. **Responsive defaults** — No fixed-width layouts unless the format requires it (device frames)

## Process Rules

- **Never output placeholder lorem ipsum** — Use meaningful placeholder text that communicates intent
- **Honest placeholders > bad implementations** — Use clear SVG wireframes instead of "almost good" designs
- **If unsure about a visual decision, ask the user** — Don't silently choose
